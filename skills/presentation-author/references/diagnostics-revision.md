# Diagnostics and Revision Policy

Diagnostics should be enabled by default.

Diagnostics may include:
- PPTX render to PNG
- contact sheet generation
- overflow validation
- font checks when enabled

Revision:
- one revision attempt by default
- no multi-pass revision
- no scoring engine
- if revision fails, preserve the initial deck
- if diagnostics pass, revision may be skipped

Do not add multi-pass quality loops unless explicitly requested in a future phase.
